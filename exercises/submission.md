# Exercise Submission Guide

Use this guide to submit all completed topic exercises.

## What to submit

1. Repository reference
   - Git branch name or repository link.
2. Completed topics list
   - Mention each file you completed under `exercises/*/excersises.md`.
3. Evidence package
   - Command outputs/log snippets.
   - Screenshots (IDE/debugger/docker where relevant).
   - Files you created (docs, notes, comparison tables).
4. Reflection
   - Biggest blocker and how you solved it.
   - One workflow improvement you will keep using.
   - One concept you still need help with.

## Quality bar before submitting

- All required exercises use Node.js + npm commands.
- Commands are reproducible from a clean clone.
- Notes are clear enough for another intern to follow.
- No secrets are committed (`.env` values are not committed).

## Optional exploration note (not graded)

After completing required npm workflow, you are encouraged to try other package managers (pnpm/yarn) and runtimes (Bun/Deno) and record differences in install speed, script compatibility, and debugging experience.
